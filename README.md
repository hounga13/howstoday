# 🔮 오늘 어때? (How's Today?) 

서버 없이 가볍게 운영하며 광고 수익을 목적으로 하는, "여기 어때?" 패러디 운세 웹페이지를 위한 기획서입니다.

---

## 📋 요구사항 명세서 (Requirements Specification)

### 1. 목표 및 타겟
- **목표**: 별도의 유지보수와 서버 비용 없이 완전한 정적 웹사이트로 동작하며, 꾸준한 트래픽을 유도해 광고 수익 창출.
- **타겟**: 매일 아침 가볍게 운세를 확인하고자 하는 10~30대 유저. 가볍고 재미있는 컨셉 선호.

### 2. 핵심 기능 (Functional Requirements)
1. **사용자 정보 입력**:
   - 사용자의 이름, 생년월일(또는 띠/별자리)을 입력받는 폼 제공.
   - 복잡한 회원가입이나 로그인 절차 일체 없음.
2. **오늘의 운세 생성 (무서버 구현)**:
   - 서버 DB 조회나 외부 유료 API를 사용하지 않습니다.
   - **구현 방식**: `현재 날짜` + `사용자의 정보(이름, 생일)`를 조합하여 시드(Seed) 값으로 쓴 후, 미리 준비된 운세 문구 풀(Pool)에서 결정론적 난수를 통해 결과를 매칭합니다. (이렇게 하면 같은 날, 같은 사용자는 항상 같은 결과를 보게 되어 신뢰성을 줍니다).
   - 운세 종류: 총운, 금전운, 연애운 등.
3. **결과 공유 (바이럴)**:
   - "친구에게 공유하고 야속한 내 운세 탓하기" 등의 재미있는 문구와 함께 카카오톡 공유 기능 및 링크 복사 기능 제공.
   - 이를 통해 추가 트래픽 직접 유도.
4. **광고 지면 (Monetization)**:
   - 운세 결과를 보기 직전, 혹은 운세 결과 하단에 Google AdSense 등 배너를 노출.

### 3. 비기능 요소 및 배포환경 (Non-functional Requirements)
- **배포 방식**: GitHub Pages, Vercel 등을 통한 100% 무료 정적 배포 (HTML, CSS, JS만 사용).
- **반응형 웹**: 철저한 **모바일 퍼스트(Mobile First)** 설계.

## 🚀 다음 마일스톤 (Next Milestones)

여행을 다녀오신 후 이어서 진행할 수 있는 **마케팅 및 기능 고도화 파이프라인** 모델입니다. 언제든 원하시는 항목을 선택해 실행할 수 있습니다.

**[Phase 2: 바이럴 성장 & 체류 시간 증대 (Engagement & Viral)]**
- [ ] **애니메이션 및 효과음 고도화**:
  - 결과창이 열릴 때 컨페티(Confetti, 폭죽) 그래픽 터트리기.
  - 운세점수를 계산하는 과정에 빠칭코 같은 다이내믹한 롤링 이펙트 구현.
- [ ] **타겟 맞춤화(띠/별자리)**:
  - 현재 입력받는 생년월일 데이터를 바탕으로 '오늘의 나의 띠/별자리' 정보를 메인 결과 상단에 함께 노출하여 개인화된 몰입감 극대화.
- [ ] **소셜 공유(카톡) 플러그인 연동**:
  - 현재 내장된 Web Share API 뿐만 아니라, `Kakao SDK`를 연동하여 카카오톡 나에게/친구에게 '커스텀 템플릿(나만의 점수가 적힌 버튼 달린 메시지)' 형태로 발송되게 최적화.

**[Phase 3: 수익화 파이프라인 완성 (Monetization)]**
- [ ] **Google AdSense 정식 연동**:
  - 여행 중에 GitHub 깃허브 페이지에 도메인이 정상적으로 캐싱되었다면, 애드센스 승인 신청 절차 진행 돕기.
  - 5초 대기 모달 창 대신, 실제 애드센스 '전면/보상형' 광고 스크립트를 올바른 위치에 삽입하기.
- [ ] **사용자 체류 분석 (Analytics)**:
  - `Google Analytics 4` (GA4) 태그를 추가하여 사용자가 어떤 운세를 가장 많이 확인하고, 어디서 많이 공유(이탈률)하는지 측정.
